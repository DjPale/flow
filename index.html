What does it do?

To understand why flow exists, you must understand the way haxe works it’s magic.

A target and a platform are not the same thing.

The haxe compiler transforms code from the haxe language, into another target language.

haxe -> javascript
haxe -> c++

It’s important to know the difference, using the js target as an example.
The javascript language, is the target.

There is no single platform associated with js, it can run in the browser, on the server, in a standalone executable on desktop or mobile. The most common platform is the web.
The web, is a platform.

With the c++ target, the code can run on a huge number of platforms, from desktop to mobile and many more. c++ is the target, Windows is the platform.

Building an application

Now that you understand the separation - the code that haxe generates is 100% ready to use, say in a webpage or an application., but has has done it’s job.

Now you need to put it all together, in a webpage or an application.

What if your c++ application haxe generates needs an icon?
What if you want to copy files across, required to run your application?
What if you want to run a script automatically before building?
What if you want to package nightly of your application for testers?

This is what flow is for
flow takes the haxe output, your project input, and creates shippable applications, games, websites or more on it’s supported platforms.

Brief Overview

flow is a project based build tool, it simply takes the flow file you describe,
and executes the tasks you request according to a flow file.

A quick overview of the available commands :

  • files - copies and templates files from your project to the build
  • icons - copies, converts and embeds icons for a build
  • package - creates a zip or tar archive from a build
  • run - builds, copies files, icons, runs hooks and launches the application

For a full list, see the usage guide.

A list of the supported platforms :

  • web - from js target (.js)
  • mac - from c++ target (.app)
  • windows - from c++ target (.exe)
  • linux - from c++ target (binary)
  • android - from c++ target (.apk)
  • ios - from c++ target (.app + xcode project)

flow has many tools, from javascript minify to interacting with the cpp build pipeline directly - it’s an expressive tool designed around flexibility and simplicity. for full details, read the flow file project format, as it’s the most direct route to learn more.

Setup guide

flow is setup through haxelib,
from a terminal (need help?)

by running the following command :

haxelib install flow

followed by

haxelib run flow setup

which will open a simple web interface get you started.

btw

  • on windows cmder is an excellent alternative to cmd.exe garbage
  • on mac/linux fish is also a really nice shell
